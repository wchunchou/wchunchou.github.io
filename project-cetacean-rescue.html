Cetacean Rescue

Cetacean Rescue is an app that help rescuers to find stranding whales or dolphins more quickly. Therefore, increase the survival rate of them. I tackled the whole UX research and design as well as in charge of the development management with the dev team.

Process

Start at 2016, this project is a three years program. One phrase a year. At the start of each year, the Taiwan Cetacean Society and I have a kick-off meeting, reflecting on the current status and setting up a new year's goal. Here's the process we went through in one phrase:

Discover

Client Interview

After the client kick-off meeting, I found that the Taiwan Cetacean Society has two primary goals for this project. First, to increase the rescue rate of whales and dolphins. Second, educating the general public of how to help with rescue when they find a stranding whale or dolphin. They already have a broad goal in mind, which they divided into three phases:

  • 2015 Phase 1: rescue report App for the public (Android Version)
  • 2016 Phase 2: iOS Version and New Feature
  • 2017 Phase 3: visualization of data

To explain my process of a project, I will focus on the first phrase, and emphasize the iteration and another two phrase at the end of this article.

User Research

With the help of TCS, I had 1x1 interview with their professional rescuer. He provided not only the insight of how rescuers work but also how the public report cases to them. Here's the key finding:

  • Rescuers
    • Rescuers need to be highly mobile, they rely on mobile phone and nearly don't use laptop or desktop PC
    • They received cases almost every two weeks
    • They work as a team. There are two active teams in Taiwan. But right now, most of them are volunteers, and their communication network needs improvement.
    • The most information they need is the location and the status of the animal.
  • General Public
    • They don't know how to act when encounter a stranding whale
    • They usually call coast guard instead of TCS.

User Story

With the key findings, I was able to transform it into user stories in a spreadsheet. User stories can not only help me create ideas but it can also serves as a great tool to create empathy with developers. Moreover, we can run the Scrum based on these user stories. A sample of user story is as following:

As a rescuer, I need the location and photos of the stranding animal so that I can bring proper tools to the right plcae.

Define

I generate idea with the user stories. Then, I create a UI flow and then mapping these ideas into each screen.

Ideas

Based on these user story, I generate some main ideas:

  • The report form should have the instruction of how to take proper photos so that the rescuers can know which tools they should bring with them.
  • The app can automatically detect location but allow the user to enter notes (ex. near Hoa-Lian Harbor) or modify their location by redetecting the location.
  • In case of spotty internet, the app will have a draft status, which they can submit later.
  • the user can receive update of their report by notification.

Project Speculation & Task Assignment

Once I know how may pages and features we need. I arranged a tech meeting with the developer head, we decided who would devleop the program and how much time we need. In addition, we were able to calculate the cost based on the estimated work hours.

Unfortunately, we had a shortage of App developer at that time. So, we outsource this part to a freelancer. From this experience, I learned a lot of how to manage a remote developer.

Design

I created the prototype with Axure RP. In this prototype, I have two challenges I want to try: First, I want to display different status of a report. That is, new report, failed report, and replied report. Second, I want to make this app looks more delightful and warm, in the opposite of suffering animal pictures that will show up in this app.

Validation

Heuristic Evaluation

When the prototype is ready, I ran a heuristic evaluation with several of my co-worker. One of the significant change accroding to the heuristic evaluation is to move the navigation tab from the bottom to the top of the screen in the Android version. They said that based on the UX research form some popular app like LINE, Android users are prone to press the wrong tab because the bottom tabs are close to the original function tabs of Android phone.


the location of tabs is different between iOS version and Android version.

Usability Test & Client Approvement

After our in-house testing, I bring the prototype to the client and ran a usbility test with them. In one hand, I would like to make some ajustiment with their involement, on the other hand, the the action also prevent them from changing spec after we go into the developing stage.

Coding

When we move into the coding process, I will create several "issues" on our issue tracking system, setting up estimated work hours and assigning them to the developers. The most difficult part of this stages would be to track their progress and help them solving difficult tasks by finding a senior developer for them. Also, I will have a daily stand-up meeting with them and explain the detail of features if they are confusing with the design.

Phase 2 and Phase 3

Phase 2 - New Feature

In phase 2, we developed the iOS version of the app, and add a new feature - Wiki. This new feature allows the user to understand different species of whales and dolphins. Also, it explained in detail of how to rescue stranded animals by taking the standard process.

Phase 3 - Visualization of Data

After collecting data for two years, we can utilize the data by visualizing them to the public. In the past, the TSC had to create a picture of a map that lists the location cases every month. It takes time and effort. By creating a website that shows the report data from the app database, we can show the most up-t-date cases with filter and interactive features.

Conclusion

This project is my first mobile app project that goes to public users. It had a more flexible schedule but a limited budget, so I was able to run this project with multiple phrases. I learned how to manage developer outside of the company and how to design a mobile UX with a long-term prospect.

One thing I think to improve is that I wish I would run task testing with more end-users. But back to 2015, I know nothing about task testing, which leaves many rooms for usability improvement
